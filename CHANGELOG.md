# История изменений (Changelog)

Все значимые изменения в этом проекте будут документированы в этом файле.

Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.0.0/),
и проект придерживается [Semantic Versioning](https://semver.org/lang/ru/).

## [4.0.0] - 2024-01-15
### Добавлено
- Полная документация всех функций в формате docstring
- Комплексные тестовые сценарии в функции `main()`
- Подробные примеры использования в документации
- Анализ сложности алгоритмов (Big O notation)
- Обработка краевых случаев (пустые строки, нестроковые типы)

### Изменено
- Рефакторинг `group_anagrams()` для лучшей читаемости
- Улучшена структура `main()` для демонстрации
- Оптимизирована проверка уникальности в `get_unique_words()`
- Сортировка результатов для предсказуемости вывода

### Архитектурные улучшения
- Четкое разделение ответственности между функциями
- Добавление модульного docstring с описанием всего модуля
- Стандартизация формата документации (Google Style)
- Добавление математического обоснования алгоритмов

## [3.0.0] - 2024-01-14
### Добавлено
- Использование `defaultdict` из collections для эффективной группировки
- Регулярные выражения для очистки текста (модуль `re`)
- Фильтрация однобуквенных слов
- Удаление дубликатов слов с использованием `set`
- Поддержка русского и английского языков

### Изменено
- Оптимизирован алгоритм `clean_text()` с использованием `re.sub()`
- Улучшена обработка различных языков (добавлена буква `ё`)
- Замена ручной обработки знаков препинания на регулярные выражения

### Удалено
- Функция `is_anagram()` - заменена на нормализацию через `normalize_word()`
- Ручной перебор знаков препинания в `clean_text()`

## [2.0.0] - 2024-01-13
### Добавлено
- Базовая реализация всех функций из заглушек
- Алгоритм нормализации слов через сортировку букв
- Простая очистка текста от знаков препинания
- Группировка анаграмм через словарь
- Базовый рабочий прототип

### Реализованные функции
- `find_anagrams_in_text()` - основная логика поиска
- `clean_text()` - ручная очистка от пунктуации
- `normalize_word()` - сортировка букв для сравнения
- `get_words_from_text()` - извлечение слов
- `is_anagram()` - проверка двух слов (позже удалена)

### Изменено
- Заглушки заменены на работающие реализации
- Добавлена базовая обработка ошибок

## [1.0.0] - 2024-01-12
### Добавлено
- Создание структуры проекта
- Заглушки всех функций с минимальными docstring
- Определение интерфейсов функций
- Базовый каркас программы
- Точка входа `if __name__ == "__main__"`

### Функции-заглушки
- `find_anagrams_in_text()` - возвращает образец данных
- `clean_text()` - возвращает входной текст без изменений
- `normalize_word()` - возвращает слово без изменений
- `get_words_from_text()` - возвращает пустой список
- `is_anagram()` - всегда возвращает `False`

## Методология разработки

### Нисходящее проектирование (Top-Down Design)
Проект разработан с использованием метода нисходящего проектирования:

1. **Итерация 1: Проектирование интерфейсов**
   - Определение сигнатур всех функций
   - Создание заглушек (stubs) с pass/return
   - Построение общей архитектуры

2. **Итерация 2: Базовая реализация**
   - Реализация основных алгоритмов
   - Простая обработка текста
   - Базовая группировка анаграмм

3. **Итерация 3: Оптимизация**
   - Использование эффективных структур данных
   - Улучшение обработки текста (регулярные выражения)
   - Добавление обработки краевых случаев

4. **Итерация 4: Production-ready версия**
   - Документирование кода
   - Добавление комплексных тестов
   - Рефакторинг для улучшения читаемости
   - Обработка ошибок
